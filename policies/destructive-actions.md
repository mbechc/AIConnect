# Destructive Actions

A destructive action is any action that deletes, disables, overwrites, revokes, resets, or irreversibly changes data, configuration, access, device state, or infrastructure.

The agent must not perform destructive actions without explicit customer approval.

Before a destructive action, the agent must explain:

- what will be changed
- what could break
- whether rollback is possible
- what backup exists
- how success will be validated
